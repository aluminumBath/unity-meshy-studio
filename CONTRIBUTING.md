# Contributing

1. Create a focused branch.
2. Keep skills concise and task-specific.
3. Do not add credentials, generated binaries, or copyrighted sample assets.
4. Run `python tools/validate_release.py`.
5. Test Unity Editor scripts in the documented Unity version.
6. Describe behavior changes and security implications in the pull request.
7. Do not weaken credit confirmation or credential safeguards.
